Objectives

As a newcomer to the world of design, I needed a way to present my designs. Learning frontend development was engaging, interesting, and exactly what I needed to get my designs out there.

The portfolio website you are about to see is a responsive, mobile first site. During the coding process, I was able to learn and grasp the multiple languages and foundations of the frontend development world.

  • Timeline: June 2023 - August 2023
  • My Role: UX/UI Designer | frontend developer
  • Tools: VS Code | Adobe XD | GitHub
  • Languages: HTML | CSS | JavaScript

Content

Jumping headfirst into a completely new world was intimidating. And hard. I loved every second of it. While HTML is the structure of the website, it is not the content. I spent some time decided what content was important to include. I did this by analyzing other portfolio sites I could find. I came away with three important pieces of information to include:

  1. A work page: People need to see my work. This lead to further deliberation about what elements of my work to include. I decided a quick, important, and fascination summary of my projects was good.
  2. An about page: I want people to know who I am! A quick summary of me, how I can be of service to employers, and the skills I have mastered.
  3. A contact section: In order for employers to contact me, I needed a contact section. Email, LinkedIn, Behance, Twitter.

Writing the Code

With the text finalized, images selected, and a rough layout in mind, I started writing. The learning curve was steep. There were so many new elements, tags, and symbols I had never seen before. With the help of my mentor from CareerFoundry, YouTube, peers, and sheer force of will, I was able to complete the layout of the site. But a bunch of elements on a page don’t look good. This where the next coding language comes in.

Design

I knew I wanted a professional site. But I didn’t want a boring site. I chose white and grey as my main background colors to keep it clean. I went for a deep blue, green, and bright green as accent colors to bring some personality into the site. Although this made the site really come together, we weren’t done quite yet.

The Code

JavaScript and CSS media queries were utilized in order to make this a mobile first site. In our world, we have many different devices with many different screen sizes. In order to make this site accessible to everyone, I used JavaScript to collapse and expand the navigation menu and CSS media queries to adjust element size and layout.

Code quality was conducted using validator.w3.org and VS Code linter plugins. Each HTML page was run through the program. I was able to locate and fix all warnings and errors that popped up. During the design process, I created my logo as an SVG. The W3 linter program picked up on some unnecessary code attached to the SVG file. I was able to locate and remedy these errors as well.

Usability Testing

Usability testing was then completed using 5 participants ages 20-60. Overall, usability test was a success. The participants enjoyed the site and found it simple and easy to understand. Multiple participants said their first impression was that they enjoyed how simple and clean the site was. The use of white space, light grey, and then a darker color (dark blue), was very popular and multiple users said it was calming.

The users were all able to complete the 4 tasks given to them. There wasn’t much of an issue with the navigation of the site. A common piece of feedback was to make the logo a navigation element back to the homepage of the app. This is a great idea for future iterations. Another common piece of feedback was the spacing of certain elements on my site. The header on the “work” page and the “relevant skills” columns were out of alignment.

The Final Product

This was a daunting challenge. I never imagined myself learning or being able to write code. The challenges and obstacles faced were ones I never thought I would need to face. Coming into a development course with zero knowledge, and ending the course having built a responsive website is exciting. The challenges were what engaged me throughout the process.

With this knowledge, I am more confident that I will be able to communicate better with development teams. I can know design with development in mind. This process has not only increased my knowledge of frontend development, but has made me a greater UX Designer as well.
